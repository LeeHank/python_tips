虛擬環境整理

1. Terminal 先開虛擬環境

  • 在 python 3.3 以後，已經有內建 venv 這個 module 了，所以現在普遍都直接用這個來建立虛擬環境
  • 指令為：
cd my_project # 先進到專案資料夾
python -m venv my_venv # python -m 表示在 terminal 要執行 python 的 module; 最後的 my_venv 自己定義
  • 做完後，專案資料夾下，會多一個 my_venv 的資料夾，裡面的 bin 資料夾底下，就有 python 等檔案。之後這個專案用的 pyhon 路徑，就是 {專案資料夾路徑}/my_venv/bin/python 。等一下啟動虛擬環境後，就可以用 which python 來驗證看看。
  • 接著啟動虛擬環境： source my_venv/bin/activate ，這就很直觀，我要去調用 my_venv 資料夾下的 bin 資料夾下的 activate 這個檔案。
  • 如果是 windows，那要改用 my_env/Scripts/activate.bat
  • 這樣就開啟虛擬環境了。

2. Jupyter lab / notebook 用這個虛擬環境

  • 在 jupyter lab 中，他的 python interpreter ，叫做 kernel 。所以，我必須先幫他註冊一個 kernel，然後這個 kernel 是我剛剛建好的虛擬環境。這樣，我跑 jupyter notebook 時，就可以選這個 kernel 來跑，就會在我的虛擬環境下跑。
  • 作法如下：
    • 在 terminal 要先進到虛擬環境中
    • 在虛擬環境下，去安裝 `pip install ipykernel` (因為在虛擬環境中，我還沒裝過這個 package ，所以記得在每個專案中，都是進到虛擬環境後，先裝這個，避不掉)
    • 執行 python -m ipykernel install --name=whatevername，這邊的意思，就是會把你現在身處的虛擬環境，註冊到 ipykernel 裡面，然後我幫這個 kernel 取個名稱(我這邊就寫了 whatevername)。所以可以發現，kernel 名稱不一定要和我現在虛擬環境的名稱一樣。但建議還是取一樣才不會搞混啦。
    • 輸入 jupyter kernelspec list ，可以看到現在被註冊過的 kernel 有哪些，裡面應該要有剛剛註冊完的 whatevername
    • 打開 jupyter lab，新增檔案時，選 whatever 的 kernel 的 notebook 檔，那這個 notebook 就會用 whatevername 這個 kernel 來執行，也就是用我剛剛的虛擬環境來執行(python 和套件都是用這個虛擬環境下的東西)
    • (然後，我之前在這邊卡一個下午，是因為我打開 notebook 後，輸入 ! which python ，結果發現根本還是調用我本機的 python，不是用虛擬環境，就氣噗噗一直重新開關jupyter lab，都搞不定。後來才發現，因為我打 ! which python，他是在背後另外開一個 terminal 來執行，所以根本就沒 activate 虛擬環境，他當然一直説他要調用本機的 python。但當我在 notebook 的 code cell 正常執行 python code (e.g. import numpy as np ，就會發現他沒有這個套件。因為，在 code cell 執行時，他是用我剛剛註冊好的 kernel 在跑，所以他乖乖的用我虛擬環境。結論：我耍笨，其實是有成功的。)
    • 最後，如果要刪除之前註冊過的 kernel ，可以這樣做： jupyter kernelspec uninstall whatevername

3. 在 VSCode 中用虛擬環境

  • 也是先 activate 後，按 ctrl + shit + p，寫 python select interpreter ，就可以選到我虛擬環境路徑下的 python，選好後，之後執行就都是用虛擬環境

4. 在 VSCode 中叫 jupyter notebook 用我的虛擬環境

  • 步驟跟 2 一樣，做完後，新增或開啟舊有的 jupyter notebook，就可以選註冊好的 kernel。
