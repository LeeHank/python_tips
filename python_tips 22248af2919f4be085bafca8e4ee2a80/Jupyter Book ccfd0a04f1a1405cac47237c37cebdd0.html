Jupyter Book

  • 先到他的官網： https://jupyterbook.org/intro.html
  • 左邊有 tutorials 可以看，選 create your first book
  • 接下來就實際走一次流程

1. 安裝

  • 第一步先安裝 jupter book ( pip install jupyter-book )

2. 開專案(導入模板)

  • 假如我想新增一個專案資料夾叫 python_ds ，放在 /Volumes/GoogleDrive/我的雲端硬碟/0. codepool_python 的下面，那我就：
    cd /Volumes/GoogleDrive/我的雲端硬碟/'0. codepool_python'
jupyter-book create python_ds/ # create 這個 jupyter-book 專案
  • 他就幫我新增好一個模板所需的所有檔案

3. python 虛擬環境處理一下

  • 在 terminal 進到這個專案資料夾裡面
cd python_ds
python -m venv python_ds_env # 新增一個虛擬環境
source python_ds_env/bin/activate # 開啟虛擬環境

# 安裝常用套件
pip install numpy pandas matplotlib sklearn

# 安裝 jupyter notebook 的 kernel 管理工具
# kernel 的意思就是 python interpreter，就是在 notebook 中，你要用哪個 python & 對應的 package 來執行 code
# 預設就是電腦最初安裝好的 python
# 但我們希望 jupyter notebook 用我們的虛擬環境，所以我現在要來幫他註冊一個 kernel
pip install ipykernel # 安裝 ipython notebook 執行時要用的 kernel 管理工具
python -m ipykernel install --name=python_ds_env # 註冊 kernel (把我現在身處的虛擬環境，加進去，到時候跑 jupyter notebook時，就可以選這個 kernel 跑，意思就是用虛擬環境的python & package 來跑) 

# 把安裝好的套件，寫到 requirements.txt
pip freeze requirements.txt

4. git 

git init
echo python_ds_env > .gitignore # 新增 .gitignore 檔案，第一行就是 python_ds_env，因為我不想把 virtual environment 也上到github上，我只要有 requirements.txt 就夠了

5. 用 jupyter lab進到專案

  • 在 terminal 中，開啟 jupyter lab
jupyter lab # 開啟 jupyter lab 服務
  • 進到專案資料夾後，把裡面的 notebooks.ipynb 這個 notebook 打開，把 kernel 改成我剛剛註冊過的 python_ds_env 。就可確保等等這個 notebook 是用虛擬環境來跑
  • 記得存檔

6. compile 出 book 來

  • 現在都還不用管內容，反正資料夾裡面的都是範本，我就先 build 出一本書就對了
  • 打開 terminal，輸入： 
cd /Volumes/GoogleDrive/我的雲端硬碟/'0. codepool_python'/python_ds # 進到專案資料夾
jupyter-book build . # 我要 build 一本書，用哪個路徑下的內容來 build? 用 `.` current資料夾

7. push 到 github

  • 在自己的 github 上，開一個 python_ds 專案，然後把東西 push 到上面去
  • 但到目前為止，還沒辦法用 github pages 的功能，來deploy我的書。因為，github pages 只吃 docs 這個資料夾下的東西，但我現在書的內容，是被 build 在 _build/html這個資料夾底下 (而他又沒辦法像 R 的 bookdown 一樣，透過改設定，把書 build 在 docs 資料夾內)
  • 所以，這時就參考官網的建議 (https://jupyterbook.org/start/publish.html)
    • 先安裝 pip install ghp-import 這個套件 (ghp = github page)
    • 直接在本地的 master 分支下，執行 ghp-import -n -p -f _build/html
  • 那他就會自動做以下事情：
    • 在本地，先建一個 gh-pages 分支
    • 只留 _build/html 的內容在根目錄，其他檔案資料夾全刪光
    • 新增一個 .nojekyll 的檔案 (就是上面 -n 所代表的意義， -n = touch .nojekyll)
    • 把這個分支，也推到 remote
  • 那現在 github 上，就有兩個分支，而我們只要切換到 gh-pages 這個分支，再去 pages 那邊設定，說我要 deploy 根目錄的內容，那就完成 deploy 了

8. 開始改內容

  • 最一開始要改的，就是 專案根目錄下的 _toc.yml 這個檔，改成像這樣的架構：
format: jb-book
root: intro
parts:
  - caption: Name of Part 1
    numbered: true
    chapters:
    - file: markdown
    - file: may_error
  - caption: Name of Part 2
    numbered: true
    chapters:
    - file: notebooks
  • 這是因為，我喜歡
    • 把章節分 block (所以才有 parts 和 caption 的設定）
    • 章節和內文標題，都要 number (所以才有 numbered: true )
    • 然後， root 所指稱的 intro，是一個 intro.md 檔 (不一定要 md 檔，你要 .ipynb 也可)，就是指書本的第一頁，要讀哪個檔案。其實就和 bookdown 裡面的 index.html 一樣意思
