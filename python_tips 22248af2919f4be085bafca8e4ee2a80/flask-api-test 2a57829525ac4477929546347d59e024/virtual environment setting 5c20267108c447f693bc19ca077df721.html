virtual environment setting

  • 開啟python專案時，最好都用 virtual environment
  • 舉例來說，我今天的專案資料夾，叫 api_test ，那我就是在terminal裡，切進這個資料夾，然後call python3 -m venv .env
    • -m 沒記錯的話，是在python環境裡下系統指令。就像在R裡寫 System(xxx) 一樣。所以，應該也可以直接在terminal裡面call venv .env 就好，不需要寫 python3 -m
    • venv 就 virtual environment 的縮寫
    • .env 這只是你的virtual environment的資料夾名稱而已。可以取任何你喜歡的名字
  • 作法和截圖如下：
# 切到專案資料夾下
cd /Volumes/GoogleDrive/我的雲端硬碟/"0. codepool_python"/flask_api_test
# 建立 virtual environment
python3 -m venv .env
# 啟動 virtual environment(進到這個環境裡面)
source .env/bin/activate
# 打開 VSCode 開始寫code
code . # . 是當前資料夾的意思，整句就是，用vscode開啟當前資料夾
  • 注意一下，當我寫 source .env/bin/activate 後，他在 terminal 的變化是： (.env) 開頭，這個意思是你已經在你定義的 .env 虛擬環境下了。那在這個環境下是什麼意思？就是我在這個環境裡用 python xxx 這個指令，他會去 .env 資料夾裡找 python 的 bin 裡有定義的指令來用。若我在這個環境下去安裝package，他會把package裝到 .env 這個資料夾下的sitepackage。若我在這個環境下 import package，他也會到 .env 這個資料夾下的 sitepackage裡面找。
  • 最後，看一下我的 code 指令，還沒被加到環境變數裡。這邊用最簡單的做法：
  • 照著做完後，把terminal重新開啟(他才會重讀系統的環境變數)，再打 code . 就可以了
  • 將 code 指令放到環境變數的做法，官網也詳細說明，可以去看看，學學別招：https://code.visualstudio.com/docs/setup/mac
  • 進到VScode後，我們會看到 flask_api_test 的專案資料夾下，多了一個 .env 的資料夾
  • 這樣就成功完成 virtual environment 的設定了
  • 這邊還要提醒很重要的一點。其實我更常在 VSCode裡面用他的terminal功能。那這時你去看 terminal 時，他使用的 python 環境還不是虛擬環境
  • 這時，只要去點左下角的 python 版本，然後選擇虛擬環境的 python，他就會正確進入到我們的虛擬環境了。
  • 順便在 terminal 裡面，做 git init，讓他受到版本控制。這我就不整理了。
